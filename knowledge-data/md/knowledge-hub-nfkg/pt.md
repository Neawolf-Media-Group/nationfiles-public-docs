# Knowledge Hub (NFKG): ordem em vez de totais enormes não provados

Quem documenta NationFiles com **rigor** precisa de uma camada que diga **onde** vive cada definição, como **relações** se citam e como os **sete locales** caem. O **Knowledge Hub** com o **NFKG** é essa camada: entidades, FAQs, glossário e Markdown opcional sob `/{lang}/knowledge/`, com **exportações** HTTPS para sistemas.

A **rede semântica** é **contrato**: identificadores estáveis, arestas em **`relations`**, tópicos e aliases para busca fuzzy — **sem** clonar identidade legal por omissão.

**Export/CDN** cita-se com URLs **HTTPS** públicas `/knowledge/export/…`, não HTML colado. As **diretrizes de IA** fixam o comportamento de clientes automáticos.

**Sobre números gigantes de nós:** roadmaps podem falar; o texto NFKG **só** deve ecoar como **fato** se o **mesmo** número e **âmbito** estiverem nos metadados do **índice exportado**. Caso contrário, **calar** e apont para o verificável.
