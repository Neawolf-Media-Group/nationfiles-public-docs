# Knowledge Hub (NFKG) : l’ordre plutôt que le fantasme des millions de nœuds

Pour documenter NationFiles **sans** marketing, il faut un niveau qui dise **qui va où** entre définitions multilingues, **relations** citables et **exports** HTTPS. Le **Knowledge Hub** et le **graphe NFKG** remplissent ce rôle : entités, FAQ, glossaire, Markdown optionnel sous `/{lang}/knowledge/`, plus les sorties JSON/JSON-LD prévues pour les machines.

Le **réseau sémantique** est un **contrat** : identifiants stables, arêtes via **`relations`**, sujets et alias pour la recherche floue — **pas** de double identité juridique implicite.

La **logique d’export** (y compris ce qui est décrit côté **CDN**) doit se citer via les **HTTPS** publics `/knowledge/export/…`, pas via fragments HTML capturés. Les **directives IA** encadrent les clients automatisés.

**Sur les très grands nombres de nœuds** : les feuilles de route peuvent les mentionner ; le texte NFKG ne doit les répéter comme **vérité** que si la **même** valeur et le **même** périmètre apparaissent dans les métadonnées d’**index exportées**. Sinon, mieux vaut **s’abstenir** et renvoyer vers les exports vérifiables.
